Running Idle: Gamified Running App

Built using: MongoDB, Express, React(w/ Redux), Node.js

When users visit Running Idle, they are met with a vintage, pixelated splash page with instructions and a sign-in/up modal. As lead backend engineer, I implemented user authnetication using Express and bcrypt to keep user passwords encyrpted within the database.

Users can create, modify and delete characters. These are analogous to "save files", each with their own set of run history, point totals, and outfit upgrades. Here, points earned from runs can be used to upgrade the character's attributes ('arms', 'legs', 'heart'). We have built in a 'cheat' button for demo purposes, so visitors can interact with character upgrades without having to actually go for a run. You're welcome.

While I was largely responsible for backend work this project, I created this entire 'character runs' page, from the Google Maps API component, to managing the relevant Redux slices of state, to creating geographically accurate database entries for each run. These runs, which each contain a time, distance, pace, and points earned, can be clicked to view the location of said run on the map. A demo run in Manhattan is available for those who either don't wish to share their location with Running Idle or those who don't wish to run.
